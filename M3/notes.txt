JSX (JavaScript XML) ka scene bas itna hai: "Ye dikhta HTML jaisa hai, lekin kaam JavaScript wala karta hai."

Simple shabdon mein, ye ek Syntactic Sugar (coding ko aasaan banane ka tareeka) hai.

Chaliye iska pura scene 3 points mein samajhte hain:

1. Ye "Bahrupiya" (Mask) hai
Jab aap React mein code likhte ho, toh aapko lagta hai aap HTML likh rahe ho:

JavaScript

const element = <h1>Hello, Amitanshu!</h1>;
Lekin Browser (Chrome/Firefox) isse nahi samajhta. Agar aap ye seedha browser ko doge, toh wo error de dega.

2. Asliyat mein kya hota hai? (Behind the Scenes)
Browser sirf JavaScript samajhta hai. Toh, Babel (ek transpiler tool) beech mein aata hai aur JSX ko Pure JavaScript mein convert kar deta hai.

Aap likhte ho (JSX):

JavaScript

// Aasaan aur readable
<h1 className="title">Hello</h1>
React/Babel isse convert karta hai isme (Asli JS):

JavaScript

// Browser ko ye milta hai
React.createElement('h1', { className: 'title' }, 'Hello');
Soch lijiye agar aapko har line ke liye React.createElement likhna padta toh kitna mushkil hota! JSX ne bas kaam aasaan kiya hai.

3. Logic + UI ka Milan
Pehle hum HTML alag file mein rakhte the aur JS alag. React ne kaha: "Websites interactive hain, toh Logic aur UI ko alag kyun rakhna?"

JSX ki wajah se aap JavaScript variables ko HTML ke beech mein ghusa sakte ho {} curly braces use karke:

JavaScript

const name = "Amitanshu";
const age = 22;

// HTML ke andar JS variable!
const element = <h1>Mera naam {name} hai aur age {age} hai.</h1>;
